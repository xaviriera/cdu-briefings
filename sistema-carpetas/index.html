01Los 3 tipos de operación

Toda operación pertenece a uno de los 3 tipos. El tipo determina dónde vive. Cada uno tiene origen, flujo y entregables distintos.

① Operación del servicio

El cliente contrata. Empieza pagando F1 (289€). Sigue a F2 si decide continuar.

Origen: contacto comercial · marketing · referido.
Flujo: F1 DIAGNÓSTICO → F2 MAP.
La carpeta padre indica la fase (F1/, F2/).
Cuando llega F3 → pasa a ③ INICIADAS.

② Operación fuera del servicio

Viene de un proveedor con comisión pactada. No paga F1, no sigue el flujo del servicio.

Decisión final: presentar como INBRUTO al Programa IN · quedárnosla nosotros · devolverla al proveedor.

③ Operación iniciada

Ya en ejecución. Equipo gestiona obra/venta. Aquí entran las que llegan a F3.

Origen doble (trazabilidad):
· DESDE_SERVICIO/ (cliente escaló al llegar F3)
· DESDE_FUERA_SERVICIO/ (nos la quedamos)

02Flujo entre los 3 tipos

Una operación nunca vive en dos sitios. Cuando cambia de tipo o de fase, su carpeta se mueve.

① SERVICIO ② FUERA SERVICIO (cliente paga F1) (proveedor con comisión) │ │ F1 → F2 EN_ANALISIS │ │ ├─ cliente acaba sin escalar a F3 ├─ derivada como INBRUTO │ → FINALIZADAS dentro de ① │ → DERIVADAS_INBRUTO dentro de ② │ │ │ cliente llega a F3 ├─ devolvemos al proveedor │ (= ejecución venta/explotación) │ → DESCARTADAS dentro de ② │ ↓ │ │ │ nos la quedamos │ │ ↓ └────────────────┐ ┌──────────────────┘ ↓ ↓ ③ OPERACIÓN INICIADA (subdividida por origen para trazabilidad) ├── DESDE_SERVICIO/ └── DESDE_FUERA_SERVICIO/

085 principios que mantienen el sistema

  1. Tres tipos, tres carpetas raíz. Ninguna op vive en dos sitios.
  2. La fase se ve en el path. Cambiar de fase = mover la carpeta padre. Conserva el material interno intacto.
  3. F3 = ejecución → INICIADAS. Cuando una op del servicio llega a F3, deja de ser servicio y pasa a iniciada.
  4. Trazabilidad del origen. Dentro de INICIADAS, DESDE_SERVICIO/ vs DESDE_FUERA_SERVICIO/.
  5. Nada se borra, todo se archiva.
